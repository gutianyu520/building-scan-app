# 建築安全評価アプリ (Building Safety Assessment App)

これはAIに基づく災害評価システムです。Reactを使用して構築されています。

## 利用可能なスクリプト

プロジェクトディレクトリで、以下を実行できます：

### `npm start`

開発モードでアプリを実行します。\
ブラウザで[http://localhost:3000](http://localhost:3000)を開いて表示します。

編集を行うと、ページはリロードされます。\
コンソールにはリントエラーも表示される場合があります。

### `npm test`

インタラクティブなウォッチモードでテストランナーを起動します。\
詳細については、[テストの実行](https://facebook.github.io/create-react-app/docs/running-tests)に関するセクションを参照してください。

### `npm run build`

`build`フォルダに本番用のアプリをビルドします。\
Reactが正しく本番モードでバンドルされ、最適なパフォーマンスのためにビルドが最適化されます。

ビルドは縮小され、ファイル名にはハッシュが含まれます。\
これでアプリはデプロイ準備完了です！

詳細については、[デプロイメント](https://facebook.github.io/create-react-app/docs/deployment)に関するセクションを参照してください。

### `npm run eject`

**注：これは一方向の操作です。一度`eject`すると、元に戻すことはできません！**

ビルドツールや設定の選択に満足していない場合は、いつでも`eject`できます。このコマンドは、単一のビルド依存関係をプロジェクトから削除します。

代わりに、すべての設定ファイルと推移的な依存関係（webpack、Babel、ESLintなど）をプロジェクトに直接コピーするため、完全に制御できます。`eject`以外のすべてのコマンドは引き続き機能しますが、コピーされたスクリプトを指すため、調整することができます。この時点で、あなたは自分自身で作業を進めることになります。

## プロジェクト構造

-   `public/`: 静的アセットと`index.html`が含まれます。
-   `src/`: Reactコンポーネントと関連ファイルが含まれます。
    -   `App.js`: メインアプリケーションコンポーネントとルーティング設定。
    -   `index.js`: アプリケーションのエントリーポイント。
    -   `style.css`: グローバルスタイル。
    -   `HomePage.js`: アプリケーションのメインランディングページ。
    -   `DisasterTypeSelectionPage.js`: 災害の種類を選択するためのページ（プレースホルダー）。
    -   `EvaluationHistoryPage.js`: 評価履歴を表示するためのページ（プレースホルダー）。
    -   `SettingsPage.js`: アプリケーション設定のためのページ（プレースホルダー）。
-   `package.json`: プロジェクトの依存関係とスクリプト。

## ホームページの機能

1.  **ロゴ**: アプリケーションのロゴ（ビルディングアイコン）。
2.  **ソフトウェア名**: 「建築安全評価」。
3.  **ソフトウェア概要**: 「AIに基づく災害評価システム」。
4.  **ステータス表示**: ネットワーク状況とGPS状況。
5.  **ナビゲーションボタン**:
    *   評価開始: 災害種類選択ページへ遷移。
    *   履歴表示: 評価履歴ページへ遷移。
    *   設定: 設定ページへ遷移。
6.  **最新の評価履歴**: 最新の2件の評価（建物名 - 評価レベル（評価日））を表示。

すべてのテキストは日本語で表示されます。
